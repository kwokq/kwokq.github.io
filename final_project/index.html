My Role: Engineer

Checklist: (from Vox Product Accessability Guidelines - Engineers)

Use the correct HTML element for your content.

Support keyboard navigation.

Understand and use HTML landmarks.

Write good alt text for your images.

Design focus states to help users navigate and understand where they are.

Help users understand inputs, and help them avoid and correct mistakes.

Use ARIA attributes when applicable.

Give users a way to skip top level navigation to access main content.

Make links descriptive.

Avoid images and iconography in pseudo-elements.

Make SVGs accessible to assistive technology.

Hide decorative elements from screen readers.

Create alternate routes for users to access information.

Links should be visually identifiable and have clear :focus and :active states.

HTML document should have a language attribute.

Suggestions

  1. Suggestion 1: Use the correct HTML element for the content

    2. The title of each category is h5 size. However, under each category, the item is h2 size, which violates the intention of header: "to create a hierarchy of headline importance." and my suggestion to this is that the title of each category should be in h2 size and the items should be in h5 size.

  2. Suggestion 2: Avoid images and iconography in pseudo-elements

    3. In the footer section, the website use many icons link. However, these icons cannot be added with description, which makes it impossible for users who use screen readers to view the website properly. Thus, those icons should be replaced by image so that descriptive words could be included for the links.

  3. Suggestion 3: Links should be visually identifiable and have clear :focus and :active states

    4. The icon links do not have clear :focus and :active states, which makes it difficult for users who use screen readers to find out that those icons are actually links. However, if :active and :focus could be set, those screen reader users could hardly miss any links.
